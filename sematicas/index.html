ETIQUETAS SEMATICAS

Las etiquetas semánticas, también conocidas como "etiquetas HTML semánticas", son una forma de marcar el contenido de una página web con información sobre su significado o propósito. Estas etiquetas proporcionan información adicional a los motores de búsqueda y a otros programas que analizan y procesan el contenido web. Algunos ejemplos de etiquetas semánticas en HTML incluyen "header", "nav", "article", "section", "aside", "footer", "main", "figure", "figcaption", entre otras. Cada una de estas etiquetas se utiliza para identificar una sección específica del contenido de la página y ayudar a los motores de búsqueda a entender cómo está estructurado el contenido. Al utilizar etiquetas semánticas apropiadas en el código HTML de una página web, se pueden mejorar la accesibilidad y la usabilidad del sitio, así como también se facilita el proceso de optimización para motores de búsqueda (SEO), ya que proporciona una mayor claridad acerca de la intención del contenido.

ETIQUETAS META

Las metaetiquetas, etiquetas meta o elementos meta son etiquetas HTML que se incorporan en el encabezado de una página web y que resultan invisibles para un visitante normal, pero de gran utilidad para navegadores u otros programas que puedan valerse de esta información. La metaetiqueta contiene información sobre el documento. Google puede interpretar un conjunto estándar de metaetiquetas. Puedes utilizar metaetiquetas personalizadas para proporcionar información adicional sobre tus páginas a Google.

¿Para que son las etiquetas sematicas?

Con las etiquetas semánticas se describe el significado del contenido y permite que los documentos HTML sean más claros para los desarrolladores y para los navegadores que procesan y renderizan sus informaciones.En este sentido, la principal ventaja de usar HTML semántico reside en que el contenido del sitio web estará correctamente estructurado. Esto, claramente, repercute en aspectos de posicionamiento, accesibilidad y relación con los motores de búsqueda, tal y como verás a continuación.Para que todas nuestras dudas se despejen, podemos echar mano de un ejemplo. Pensemos en la etiqueta STRONG ¿Qué nos indica? Que un texto debe resaltarse con mayor fuerza con respecto a otros textos que no sean "strong", parecido a lo que nos indica la etiqueta B, que todavía es más concreta, al especificar que un texto debe ser colocado en negrita. Pensemos ahora en una etiqueta DIV, o P. La primera es una división y P es un párrafo. Todo esto no son más que maneras de decirle al navegador cómo debe formatear una página, qué debe separar en un texto, en una división normal, o en un párrafo, etc.
